# API 端点文档

本文档描述了 God's Bookmark 的所有 API 端点。所有需要认证的端点都要求用户已登录。

## 认证端点（旧版，向后兼容）

### 管理员登录（旧版）
- **端点**：`POST /api/login`
- **说明**：管理员登录（旧版 API，向后兼容）
- **请求体**：
  ```json
  {
    "username": "admin",
    "password": "admin"
  }
  ```
- **响应**：返回认证 Cookie

### 登出（旧版）
- **端点**：`POST /api/logout`
- **说明**：登出（旧版 API，向后兼容）
- **响应**：清除认证 Cookie

### 检查登录状态（旧版）
- **端点**：`GET /api/check-auth`
- **说明**：检查登录状态（旧版 API，向后兼容）
- **响应**：返回当前登录状态

## 用户管理端点

### 用户登录
- **端点**：`POST /api/users/login`
- **说明**：用户登录
- **请求体**：
  ```json
  {
    "username": "username",
    "password": "password"
  }
  ```
- **响应**：返回认证 Cookie 和用户信息

### 用户登出
- **端点**：`POST /api/users/logout`
- **说明**：用户登出
- **响应**：清除认证 Cookie

### 检查用户登录状态
- **端点**：`GET /api/users/check-auth`
- **说明**：检查当前用户登录状态
- **响应**：返回用户信息或未认证状态

### 获取用户列表
- **端点**：`GET /api/users`
- **说明**：获取所有用户列表（需管理员权限）
- **权限**：管理员
- **响应**：返回用户列表数组

### 创建用户
- **端点**：`POST /api/users`
- **说明**：创建新用户（需管理员权限）
- **权限**：管理员
- **请求体**：
  ```json
  {
    "username": "newuser",
    "password": "password",
    "role": "user"
  }
  ```
- **响应**：返回创建的用户信息

### 更新用户
- **端点**：`PUT /api/users/:id`
- **说明**：更新用户信息（管理员可更新任何用户，普通用户只能更新自己的密码）
- **权限**：管理员可更新任何用户，普通用户只能更新自己
- **请求体**：
  ```json
  {
    "password": "newpassword"
  }
  ```
- **响应**：返回更新后的用户信息

### 删除用户
- **端点**：`DELETE /api/users/:id`
- **说明**：删除用户（需管理员权限）
- **权限**：管理员
- **响应**：返回删除结果

## 书签端点

### 获取所有书签
- **端点**：`GET /api/bookmarks`
- **说明**：获取当前用户的所有书签（需认证）
- **权限**：已登录用户
- **响应**：返回书签树结构（JSON）

### 保存书签
- **端点**：`POST /api/bookmarks`
- **说明**：保存书签数据（需认证）
- **权限**：已登录用户
- **请求体**：
  ```json
  {
    "bookmarks": { /* 书签树结构 */ }
  }
  ```
- **响应**：返回保存结果

### 记录书签点击
- **端点**：`POST /api/bookmark/click`
- **说明**：记录书签点击次数（用于 Top10 统计）
- **权限**：已登录用户
- **请求体**：
  ```json
  {
    "url": "https://example.com"
  }
  ```
- **响应**：返回更新结果

### 同步单个书签（扩展用）
- **端点**：`POST /api/bookmark/sync`
- **说明**：同步单个书签到服务器（浏览器扩展使用）
- **请求体**：
  ```json
  {
    "category": "分类名称",
    "bookmark": {
      "name": "书签名称",
      "url": "https://example.com",
      "icon": "图标URL"
    },
    "action": "created|updated|removed"
  }
  ```
- **响应**：返回同步结果

### 批量同步书签（扩展用）
- **端点**：`POST /api/bookmark/sync-all`
- **说明**：批量同步书签（浏览器扩展使用）
- **请求体**：
  ```json
  {
    "bookmarks": [ /* 书签数组 */ ]
  }
  ```
- **响应**：返回同步结果

### 同步文件夹操作（扩展用）
- **端点**：`POST /api/bookmark/sync-folder`
- **说明**：同步文件夹操作（创建/删除文件夹）
- **请求体**：
  ```json
  {
    "category": "文件夹名称",
    "action": "created|removed"
  }
  ```
- **响应**：返回同步结果

### 检查书签是否存在（扩展用）
- **端点**：`GET /api/bookmark/check-exists`
- **说明**：检查书签是否已存在（浏览器扩展使用）
- **查询参数**：`?url=https://example.com`
- **响应**：返回是否存在

### 获取所有书签（扩展用）
- **端点**：`GET /api/bookmark/get-all`
- **说明**：获取所有书签（浏览器扩展使用）
- **响应**：返回书签列表

### 获取热门书签
- **端点**：`GET /api/bookmark/top`
- **说明**：获取当前用户的热门书签（Top10）
- **权限**：已登录用户
- **响应**：返回 Top10 书签列表

### 清除所有用户的 Top10 数据
- **端点**：`DELETE /api/bookmark/top/all`
- **说明**：清除所有用户的 Top10 统计数据（需管理员权限）
- **权限**：管理员
- **响应**：返回清除结果

## 认证说明

大部分 API 端点需要用户认证。认证方式：

1. **登录**：通过 `POST /api/users/login` 获取认证 Cookie
2. **携带 Cookie**：后续请求自动携带 Cookie 进行认证
3. **登出**：通过 `POST /api/users/logout` 清除认证 Cookie

## 权限说明

- **普通用户**：可以访问和管理自己的数据
- **管理员**：可以访问所有用户数据，管理用户账户

## 错误响应

所有 API 在出错时返回标准错误格式：

```json
{
  "error": "错误描述",
  "code": "错误代码"
}
```

常见 HTTP 状态码：
- `200` - 成功
- `400` - 请求错误
- `401` - 未认证
- `403` - 权限不足
- `404` - 资源不存在
- `500` - 服务器错误
