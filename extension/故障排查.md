# 书签同步故障排查指南

## 问题：点击"立即同步所有书签"后同步失败

### 排查步骤

#### 1. 检查扩展后台日志

1. 打开扩展管理页面：`chrome://extensions/` 或 `edge://extensions/`
2. 找到"God's Bookmark 书签同步"扩展
3. 点击"查看视图" → "背景页"（或"service worker"）
4. 查看控制台日志，查找以下信息：
   - `[书签同步] 收到消息: {action: 'syncAll'}`
   - `[书签同步] 开始执行同步请求...`
   - `[书签同步] 开始同步所有书签...`
   - `[书签同步] 找到 X 个书签`
   - 任何错误信息

#### 2. 检查弹出窗口日志

1. 右键点击扩展图标 → "检查弹出窗口"
2. 在开发者工具的控制台中查看日志：
   - `[弹出窗口] 发送同步请求...`
   - `[弹出窗口] 收到响应: {...}`
   - 任何错误信息

#### 3. 检查服务器日志

在运行 `node server.js` 的终端窗口中，查看：
- `[书签同步-批量] 同步完成: ...`
- 任何错误信息

#### 4. 常见问题及解决方法

##### 问题 1: "无法连接到服务器"
**原因**: 服务器未运行或地址不正确

**解决方法**:
1. 确认服务器正在运行：`node server.js`
2. 在扩展弹出窗口中点击"测试连接"
3. 确认服务器地址正确（默认：http://localhost:3000）

##### 问题 2: "HTTP 400: Invalid bookmarks data format"
**原因**: 发送的数据格式不正确

**解决方法**:
1. 查看扩展后台日志中的"请求数据大小"
2. 检查是否有书签数据
3. 重新加载扩展后再试

##### 问题 3: "没有找到书签"
**原因**: 浏览器书签栏为空，或无法访问书签树

**解决方法**:
1. 确认浏览器书签栏中有书签
2. 检查扩展权限：扩展管理页面 → 确认"书签"权限已授予
3. 尝试在浏览器中添加一个测试书签后再同步

##### 问题 4: 消息传递失败
**原因**: popup 和 background 之间的消息通道问题

**解决方法**:
1. 重新加载扩展
2. 关闭并重新打开弹出窗口
3. 检查是否有 JavaScript 错误

##### 问题 5: 同步部分成功但书签未显示
**原因**: 网站页面未刷新或数据未正确保存

**解决方法**:
1. 刷新网站页面（Ctrl+F5 强制刷新）
2. 检查服务器日志确认数据已保存
3. 在网站中手动点击"设置"查看书签是否已保存

### 调试命令

在扩展后台页面的控制台中，可以手动执行：

```javascript
// 测试同步功能
syncAllBookmarks().then(console.log).catch(console.error);

// 检查配置
getConfig().then(console.log);

// 测试服务器连接
fetch('http://localhost:3000/api/check-auth').then(r => r.json()).then(console.log);

// 获取书签树
chrome.bookmarks.getTree().then(console.log);
```
