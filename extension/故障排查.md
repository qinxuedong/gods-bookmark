# 扩展故障排查

## 1. 扩展已加载，但同步没有生效

检查：

- 扩展是否已启用
- 服务器地址是否填写正确
- 当前账号是否已登录网站
- 同步相关开关是否开启

建议：

1. 先执行一次“立即同步所有书签”
2. 刷新网站页面
3. 查看扩展后台日志和服务端日志

## 2. 书签新增或删除没有同步

检查：

- `syncOnCreate` / `syncOnRemove` 是否开启
- 网站页面是否在线打开
- 服务端是否已重启到最新版本

建议：

1. 先执行一次“立即同步所有书签”
2. 再测试新增或删除
3. 检查服务端日志是否收到同步请求

## 3. 浏览器批量移动后网页位置不一致

当前版本已在移动后自动触发一轮全量对账。

如果仍不一致：

1. 确认扩展已重新加载
2. 确认服务端已重启
3. 再执行一次“立即同步所有书签”

## 4. 合并书签后浏览器里还有残留重复项

当前版本“合并书签”是手动工具，入口在网站控制中心。

检查：

- 合并规则是否满足“名称 + URL 同时相同”
- 扩展是否已重新加载到最新版本
- 网站操作后浏览器是否仍在线

建议：

1. 点击控制中心中的“合并书签”
2. 合并完成后等待浏览器侧全量对账结束
3. 如果仍有残留，再执行一次“立即同步所有书签”

## 5. favicon 不显示

当前版本统一走服务器 `/api/favicon`。

检查：

- 服务端是否已重启
- 服务器地址是否配置正确
- 浏览器是否能访问：

```text
http://你的服务器地址/api/favicon?url=https%3A%2F%2Fexample.com
```

## 6. Ctrl+Space 没有打开搜索

检查：

- 浏览器快捷键是否被其他插件占用
- 扩展是否已重新加载
- 当前网页是否为普通 `http/https` 页面

建议：

1. 打开扩展管理页重新加载扩展
2. 刷新当前网页
3. 再按一次 `Ctrl+Space`

## 7. 发布前检查

建议至少完成：

1. 普通网页与书签首页中的 `Ctrl+Space`
2. 新增、删除、移动书签与文件夹同步
3. 双设备在线实时同步
4. “合并书签”手动执行
5. 远程部署中 favicon 显示
