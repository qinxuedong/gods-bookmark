# God's Bookmark 扩展故障排查

## 1. 无法连接服务器

检查项：

- 网站服务是否已经启动
- 扩展中的服务器地址是否填写正确
- 本机与服务器之间的网络和防火墙是否允许访问

建议操作：

1. 在浏览器里直接访问服务器地址
2. 打开扩展弹窗，点击“测试连接”
3. 如刚修改服务端代码，请先重启服务

## 2. 书签新增或删除没有同步

检查项：

- 扩展是否已重新加载
- 自动同步相关开关是否开启
- 网站页面是否已经登录且保持在线

建议操作：

1. 先执行一次“立即同步所有书签”
2. 打开 `chrome://extensions/` 查看扩展后台日志
3. 检查服务端日志是否收到同步请求

## 3. 另一台设备没有实时更新

实时同步依赖网站页面在线接收广播。

建议操作：

1. 两台设备都打开 God's Bookmark 页面并保持登录
2. 两台设备都重新加载扩展
3. 在设备 A 新增或删除书签后，观察设备 B 是否自动更新
4. 如果仍未更新，刷新设备 B 的网站页面再测一次

## 4. 书签图标不显示

当前版本统一走服务器 `/api/favicon`。

检查项：

- 服务端是否已更新到最新代码
- 服务端是否已重启
- 服务器地址配置是否正确

建议操作：

1. 重新加载扩展
2. 刷新网站页面
3. 在浏览器直接访问：

```text
http://你的服务器地址/api/favicon?url=https%3A%2F%2Fexample.com
```

如果该地址无法返回图片，请优先排查服务端。

## 5. Ctrl+Space 没有打开搜索

检查项：

- 当前焦点是否在输入框、文本域或可编辑区域内
- 网页是否刚加载完成
- 扩展是否已被重新加载但页面未刷新

建议操作：

1. 先刷新当前网页
2. 再按一次 `Ctrl+Space`
3. 如果仍不生效，重新加载扩展后再刷新页面

## 6. 控制台出现“Extension context invalidated”

这通常不是业务错误，而是扩展刚被重新加载后，旧页面里的 content script 失效。

处理方式：

1. 重新加载扩展
2. 刷新当前网页
3. 再次测试同步或搜索功能

## 7. 发布前检查

可运行以下命令做基础校验：

```bash
node --check extension/background.js
node --check extension/content.js
node --check extension/content-search.js
node --check extension/popup.js
node --check extension/options.js
node --check extension/search-modal-injected.js
npm run build-extension
```

如果打包成功，再检查 `extension.zip` 根目录中是否直接包含 `manifest.json`。
