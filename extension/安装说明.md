# God's Bookmark 书签同步扩展 - 安装说明

## 📦 安装步骤

### 1. 准备扩展文件

确保 `extension` 文件夹中包含以下文件：
- `manifest.json`
- `background.js`
- `popup.html`
- `popup.js`
- `options.html`
- `options.js`
- `icon16.png`, `icon48.png`, `icon128.png`（图标文件，可选）

### 2. 安装扩展（Chrome）

1. 打开 Chrome 浏览器
2. 在地址栏输入 `chrome://extensions/` 并回车
3. 打开右上角的"开发者模式"开关
4. 点击左上角的"加载已解压的扩展程序"
5. 选择项目的 `extension` 文件夹
6. 扩展安装完成！

### 3. 安装扩展（Edge）

1. 打开 Edge 浏览器
2. 在地址栏输入 `edge://extensions/` 并回车
3. 打开左侧的"开发人员模式"开关
4. 点击"加载解压缩的扩展"
5. 选择项目的 `extension` 文件夹
6. 扩展安装完成！

## ⚙️ 配置使用

### 首次配置

1. **启动服务器**
   ```bash
   node server.js
   ```
   确保服务器在 `http://localhost:3000` 运行

2. **配置扩展**
   - 点击浏览器工具栏的扩展图标
   - 在弹出窗口中确认服务器地址（默认：`http://localhost:3000`）
   - 点击"测试连接"确认服务器正常
   - 勾选"启用自动同步"
   - 点击"保存设置"

3. **开始使用**
   - 在浏览器中添加书签时，会自动同步到网站
   - 书签会根据浏览器书签栏的文件夹自动分类
   - 根目录的书签会保存到"浏览器书签"分类

## 🎯 功能说明

### 自动同步模式

- ✅ **新书签时同步**（默认开启）：在浏览器中添加书签时自动同步
- ⚙️ **书签更新时同步**（可选）：修改书签名称或URL时同步
- ⚙️ **书签删除时同步**（可选）：删除书签时同步删除

### 分类规则

- **文件夹书签**：保存在浏览器书签栏文件夹中的书签，会按文件夹名称创建分类
- **根目录书签**：直接保存在书签栏根目录的书签，会保存到"浏览器书签"分类

### 示例

假设您的浏览器书签结构如下：
```
书签栏
├── 工作
│   ├── GitHub
│   └── Jira
├── 学习
│   ├── MDN
│   └── Stack Overflow
└── Google（根目录）
```

同步后会创建：
- 分类"工作"：包含 GitHub、Jira
- 分类"学习"：包含 MDN、Stack Overflow
- 分类"浏览器书签"：包含 Google

## 🔧 高级设置

点击扩展图标 → "高级设置"，可以配置：
- 服务器地址
- 默认分类名称
- 同步选项（创建/更新/删除）
- 启用/禁用自动同步

## ❓ 常见问题

### Q: 扩展无法连接服务器？
A: 
1. 确认服务器正在运行（`node server.js`）
2. 检查服务器地址是否正确
3. 确认防火墙允许本地连接

### Q: 书签没有同步？
A:
1. 检查扩展是否启用（扩展管理页面）
2. 确认"启用自动同步"已勾选
3. 确认对应的同步选项已启用（如"新书签时同步"）
4. 查看扩展后台日志（扩展管理页面 → 查看视图 → 背景页）

### Q: 书签同步到错误的分类？
A:
- 确认书签添加到了正确的文件夹
- 根目录的书签会保存到"浏览器书签"分类（可在设置中修改）

### Q: 重复书签问题？
A:
- 相同URL的书签会被更新而不是重复添加
- 如果同一URL在不同分类，会分别添加

## 📝 注意事项

1. **服务器运行**：扩展需要服务器运行才能同步书签
2. **网络访问**：扩展会访问本地服务器（localhost:3000），请确保防火墙允许
3. **权限**：扩展需要书签访问权限，安装时会提示
4. **数据安全**：书签数据仅在本地服务器存储，不会上传到第三方

## 🐛 调试方法

1. **查看扩展日志**：
   - 打开 `chrome://extensions/` 或 `edge://extensions/`
   - 找到扩展，点击"查看视图" → "背景页"
   - 查看控制台日志

2. **检查服务器日志**：
   - 查看运行 `node server.js` 的终端窗口
   - 查看是否有 API 请求日志

3. **测试API**：
   ```bash
   curl -X POST http://localhost:3000/api/bookmark/sync \
     -H "Content-Type: application/json" \
     -d '{"category":"测试","bookmark":{"name":"测试","url":"https://example.com","icon":"🔗"},"action":"created"}'
   ```
