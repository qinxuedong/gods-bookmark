## Cron 表达式基础

Cron 表达式是一个字符串，用于定义定时任务的执行时间。格式如下：

```
分钟 小时 日期 月份 星期
```

**格式说明**：5个字段，用空格分隔，从左到右分别表示：
- **分钟** (0-59)
- **小时** (0-23)
- **日期** (1-31)
- **月份** (1-12)
- **星期** (0-7，其中 0 和 7 都表示星期日)

---

## 字段说明

### 1. 分钟 (Minutes)
- **取值范围**：0-59
- **示例**：
  - `0` - 整点（每小时的第0分钟）
  - `30` - 每小时的第30分钟
  - `*/15` - 每15分钟
  - `0,30` - 每小时的第0分钟和第30分钟

### 2. 小时 (Hours)
- **取值范围**：0-23（24小时制）
- **示例**：
  - `0` - 凌晨0点（午夜）
  - `2` - 凌晨2点
  - `14` - 下午2点
  - `*/6` - 每6小时
  - `9-17` - 上午9点到下午5点

### 3. 日期 (Day of Month)
- **取值范围**：1-31
- **示例**：
  - `1` - 每月1号
  - `15` - 每月15号
  - `1,15` - 每月1号和15号
  - `*/5` - 每5天
  - `1-10` - 每月1号到10号

### 4. 月份 (Month)
- **取值范围**：1-12
- **示例**：
  - `1` - 1月
  - `12` - 12月
  - `*/3` - 每3个月
  - `1,4,7,10` - 1月、4月、7月、10月（季度）

### 5. 星期 (Day of Week)
- **取值范围**：0-7（0和7都表示星期日）
  - `0` 或 `7` = 星期日
  - `1` = 星期一
  - `2` = 星期二
  - `3` = 星期三
  - `4` = 星期四
  - `5` = 星期五
  - `6` = 星期六
- **示例**：
  - `0` - 每个星期日
  - `1` - 每个星期一
  - `1-5` - 周一到周五（工作日）
  - `0,6` - 周六和周日（周末）

## 特殊字符

### `*` (星号)
- **含义**：匹配所有可能的值
- **示例**：
  - `* * * * *` - 每分钟执行
  - `0 * * * *` - 每小时的第0分钟执行

### `,` (逗号)
- **含义**：指定多个值
- **示例**：
  - `0 2,14 * * *` - 每天凌晨2点和下午2点
  - `0 2 1,15 * *` - 每月1号和15号的凌晨2点

### `-` (连字符)
- **含义**：指定范围
- **示例**：
  - `0 9-17 * * *` - 每天上午9点到下午5点的整点
  - `0 2 1-10 * *` - 每月1号到10号的凌晨2点

### `/` (斜杠)
- **含义**：指定间隔
- **示例**：
  - `*/15 * * * *` - 每15分钟
  - `0 */6 * * *` - 每6小时
  - `0 2 */3 * *` - 每3天的凌晨2点

### `?` (问号)
- **含义**：不指定值（在某些系统中使用，本系统不支持）

|------------|------|
| `0 0 * * *` | 每天午夜（00:00） |

### 每周执行

|------------|------|

| `0 2 * * 5` | 每周五凌晨2点 |
| `0 2 * * 1-5` | 每周一到周五凌晨2点（工作日） |
| `0 2 * * 0,6` | 每周六和周日凌晨2点（周末） |

| Cron 表达式 | 说明 |
|------------|------|
| `0 2 1 * *` | 每月1号凌晨2点 |

| `0 2 1,15 * *` | 每月1号和15号凌晨2点 |

|------------|------|

|------------|------|

| `0 */6 * * *` | 每6小时整点 |

## 注意事项

### ⚠️ 重要提示

1. **时区设置**
   - Cron 表达式使用服务器的本地时区
   - 请确保服务器时区设置正确

2. **日期和星期的冲突**
   - 当同时指定日期和星期时，两者是"或"的关系
   - 例如：`0 2 1 * 0` 表示每月1号**或**每个星期日

3. **月份天数限制**
   - 2月只有28/29天，使用 `31` 的表达式在2月不会执行
   - 建议使用 `1-28` 或 `1-30` 来避免问题

4. **备份频率建议**
   - **高频率数据**：建议每天备份（`0 2 * * *`）
   - **中等频率数据**：建议每周备份（`0 2 * * 0`）
   - **低频率数据**：建议每月备份（`0 2 1 * *`）

5. **测试表达式**
   - 设置后建议先手动测试一次
   - 检查备份历史确认定时任务是否正常执行

6. **服务器时间**
   - 确保服务器系统时间准确
   - 建议使用 NTP 同步时间

## 常见问题 FAQ

### Q1: 为什么我的备份没有按时执行？
**A:** 请检查：
1. 备份配置是否已启用
2. Cron 表达式格式是否正确
3. 服务器时间是否准确
4. 查看服务器日志是否有错误信息

### Q2: 可以设置多个备份时间吗？
**A:** 可以，使用逗号分隔多个时间点：
- `0 2,14 * * *` - 每天凌晨2点和下午2点

### Q3: 如何设置每小时的备份？
**A:** 使用以下表达式：
- `0 * * * *` - 每小时整点
- `30 * * * *` - 每小时的第30分钟

### Q4: 如何设置每5分钟备份一次？
**A:** 使用：`*/5 * * * *`
⚠️ **注意**：过于频繁的备份可能影响系统性能，建议至少间隔15分钟以上。

### Q5: 如何取消自动备份？
**A:** 在备份配置中：
1. 选择"不自动备份（仅手动）"
2. 或者清空 Cron 表达式字段
3. 或者取消勾选"启用此备份配置"
